# JMComic 插件 for MaiBot

基于 JMComic-Crawler-Python 的禁漫天堂漫画插件，支持漫画搜索、章节下载、图片发送等功能。

## ⚠️ 重要警示

**群聊环境中使用图片发送功能存在极高风险！**

- 在群聊中发送大量图片极容易被平台检测并导致账号封禁
- 强烈建议仅在**私聊环境**中使用图片发送功能
- 如需在群聊中使用，强烈建议使用 PDF 模式发送（命令中添加 `pdf` 参数）

## 功能特性

- 📚 **漫画搜索** - 通过关键字搜索禁漫天堂的漫画
- 📥 **章节下载** - 下载指定漫画的特定章节
- 🖼️ **图片发送** - 支持合并转发或逐张发送漫画图片
- 📄 **PDF 导出** - 可将漫画章节合并为 PDF 文件发送（推荐用于群聊）
- 📦 **PDF 压缩** - 自动压缩 PDF 文件大小，增加成功发送概率
-  **自动依赖安装** - 首次使用时自动安装所需 Python 包
- ⚙️ **灵活配置** - 支持代理、域名、认证等自定义配置

## 安装说明

### 自动安装（推荐）

插件已内置自动依赖安装功能，首次使用时会自动检测并安装所需的 Python 包：
- `jmcomic` - 禁漫天堂爬虫库
- `reportlab` - PDF 生成库
- `Pillow` - 图片处理库

### 手动安装

如需手动安装依赖，可执行以下命令：

```bash
pip install jmcomic reportlab Pillow
```

## 使用方法

### 基本命令

```
/jm <漫画ID> [章节号] [pdf/jpg] - 下载并发送漫画
/jm help - 查看指令
```

### 参数说明

- `<漫画ID>`: 必填，禁漫天堂漫画的 ID 号码
- `[章节号]`: 可选，指定下载第几章，默认为第 1 章
- `[pdf/jpg]`: 可选，指定输出格式：
  - `pdf` - 将图片合并为 PDF 文件发送（默认，推荐用于群聊）
  - `jpg` - 以图片格式逐张发送或合并

### 使用示例

```
/jm 123456         - 下载漫画 123456 的第 1 章（PDF格式）
/jm 123456 2       - 下载漫画 123456 的第 2 章（PDF格式）
/jm 123456 pdf     - 下载漫画 123456 的第 1 章，PDF格式
/jm 123456 2 pdf   - 下载漫画 123456 的第 2 章，PDF格式
/jm 123456 2 jpg   - 下载漫画 123456 的第 2 章，图片格式
/jm help           - 显示帮助信息
```

## 配置说明

插件配置文件位于 `config.toml`，支持以下配置项：

### 基础配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `plugin.enabled` | 是否启用插件 | `true` |
| `components.enable_command` | 是否启用命令 | `true` |

### API 配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `api.domain` | JMComic 自定义域名 | `""` |
| `api.proxy` | 代理地址 | `""` |

### 认证配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `auth.username` | JMComic 用户名 | `""` |
| `auth.password` | JMComic 密码 | `""` |

### 功能配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `features.cooldown_seconds` | 命令冷却时间（秒） | `30` |
| `features.max_pages_per_send` | 单次最多发送页数 | `1000` |
| `features.use_forward_message` | 使用合并转发 | `true` |
| `features.images_per_pdf` | 每个PDF包含的图片数量（0表示不限制，最大60） | `40` |

## 安全建议

### 群聊环境配置

如果必须在群聊中使用，请务必在命令中添加 `pdf` 参数：（默认发送pdf）

```
/jm 123456 pdf
```

这样可以大幅降低被平台检测的风险。

### 私聊环境配置

私聊环境中可以安全使用图片发送功能：

```
/jm 123456 jpg
```

或使用合并转发模式：

```toml
[features]
use_forward_message = true
```

## 技术特性

### 自动依赖管理

插件采用延迟加载和自动安装机制：
- 依赖包仅在首次使用时安装
- 安装状态被缓存，避免重复安装
- 支持 PIL (Pillow) 等特殊包名映射

### 智能下载

- 支持多种章节下载方式
- 自动搜索下载文件位置
- 下载完成后自动清理临时文件

### 灵活的发送方式

- **合并转发模式**：将所有图片打包在一条合并转发消息中
- **逐张发送模式**：逐条发送每张图片
- **PDF 模式**：将图片合并为 PDF 文件发送（推荐用于群聊）
- **PDF 压缩**：自动压缩图片分辨率（最大1500px）和质量（JPEG 75%），减小文件大小

### PDF 分卷功能

当图片数量较多时，PDF 会自动分卷：
- 默认每个 PDF 包含 40 张图片（可通过 `images_per_pdf` 配置，范围 0-60）
- 如果图片总数小于 60 张，则合并为一个 PDF
- 设置 `images_per_pdf = 0` 可禁用分卷，所有图片放入一个 PDF

## 注意事项

1. 请遵守禁漫天堂的使用条款和版权规定
2. 合理使用命令冷却时间，避免对服务器造成压力
3. 大量下载时建议使用 PDF 模式以减少消息数量
4. 如遇网络问题，可配置代理加速访问
5. **群聊环境强烈建议使用 PDF 模式，避免账号被封禁**
